### Phase 3: Server-Side Aggregation

**Move Data Fetching to Edge Function**:
```typescript
// Instead of accepting reportData from client,
// fetch raw data server-side (RLS-filtered)

const { data: deals } = await supabaseAdmin
  .from('deals')
  .select('*')
  .eq('org_id', caller.org_id)
  // RLS filters to app_manages() automatically

// Aggregate stats
const reportData = {
  totalDeals: deals.length,
  activeDeals: deals.filter(d => !['closed_won', 'closed_lost'].includes(d.stage)).length,
  closedWon: deals.filter(d => d.stage === 'closed_won').length,
  closedLost: deals.filter(d => d.stage === 'closed_lost').length,
  pipelineValue: deals.filter(d => d.stage !== 'closed_won' && d.stage !== 'closed_lost')
    .reduce((sum, d) => sum + (d.value || 0), 0),
  wonValue: deals.filter(d => d.stage === 'closed_won')
    .reduce((sum, d) => sum + (d.value || 0), 0)
}
```

**Benefits**:
- More secure (client can't fake stats)
- Consistent with RLS (guaranteed filtered correctly)
- Simpler client code
